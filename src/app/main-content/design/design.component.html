Main Screen
Tap to open
The application needs to be as simple as possible. Therefore, after launching it the first thing users will see is a chart showing their expenses for the current month. Underneath it there will be quick access buttons which will enable the user to add new expense, add new goal or view statistics. Main view will also consist of a small table showing latest transactions, with a “View All” button leading to transaction history search screen. Menu
Tap to open
A very simple, standardised menu with options leading to different sections of the application. One of the functionalities of the application is to display undescribed records, thus the menu will display such transactions as a reminder. Statistics
Tap to open
Statistics section will display a simple pie chart showing the expenses for given categories, previously registered by the user. It will also display a more complicated Month Comparison bar chart to present the expenses over the “do not spend more than” goal expectation for current and previous months. Transaction Details
Tap to open
Transaction details view will contain a map on which users will be able to see where they made a selected transaction. It will also display some details about the transaction such as value registered, time and date, description and selected category of transaction. There will be also ability to edit each transaction. Transaction History
Tap to open
Transaction history – a simple filtered table where users will be able to search for transactions they made in the past. It will consist of simple data such as value and category. Articles
Tap to open
Articles section will consist of live thread financial news and articles for users to read. This section is designed for users who are more caring about the financial world. It may also persuade other users to start gaining interest about it.
